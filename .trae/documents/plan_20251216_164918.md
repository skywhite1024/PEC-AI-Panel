# 饼图颜色问题修复计划

## 问题分析
饼图全部显示为蓝色，是因为三个圆形元素使用了`fill`属性填充，导致最后绘制的蓝色圆形（效率）完全覆盖了前面的两个圆形（体积和成本）。

## 解决方案
将所有圆形的`fill`属性设置为透明，使用`stroke`属性来显示扇形颜色，并调整`stroke-width`以确保扇形能够完全覆盖圆形区域。

## 实现步骤

### 1. 修改 SVG 圆形属性
在 `PanelComponents.tsx` 中修改 `CustomPieChart` 组件的 SVG 部分：

- 将所有圆形的 `fill` 属性改为 `transparent`
- 将原有的 `fill` 颜色值移到 `stroke` 属性
- 将 `stroke-width` 从 1 增加到 100（等于圆的直径），确保扇形完全覆盖圆形
- 移除白色描边（`stroke="white"`），因为不再需要

### 2. 调整扇形绘制顺序
保持原有的绘制顺序：
1. 体积（灰色）- 0 到 volLength
2. 成本（浅紫色）- volLength 到 volLength+costLength
3. 效率（蓝色）- volLength+costLength 到 circumference

### 3. 测试效果
- 确保饼图显示三种不同颜色
- 验证扇形大小随输入值动态变化
- 保持标签位置正确

## 文件修改
- `components/PanelComponents.tsx` - 修改 `CustomPieChart` 组件的 SVG 部分

## 技术细节
- 使用 `stroke` 属性实现扇形颜色，`stroke-dasharray` 和 `stroke-dashoffset` 实现扇形大小
- `stroke-width` 设置为 100（直径），确保扇形完全覆盖圆形
- 保持原有计算逻辑不变，只修改 SVG 渲染方式
