# 实现用户注册与登录模块计划

## 1. 项目分析
- 项目使用 React + TypeScript + Vite
- 样式风格：科技感/工业风，主色调为 #5B5FC7（紫色）
- 现有组件使用 Tailwind CSS 进行样式设计
- 没有使用 React Router，所有内容在 App.tsx 中通过状态管理
- 现有用户信息显示在侧边栏底部，需要替换为登录状态

## 2. 实现计划

### 2.1 接口实现
- 在 `services/api.ts` 中添加 Auth 相关类型定义
- 实现登录、注册和发送验证码的 Mock 接口
- 定义清晰的请求和响应类型

### 2.2 组件开发
- 在 `components` 目录下创建 `Auth` 文件夹
- 开发 `AuthModal.tsx` 组件，包含：
  - 登录/注册 Tab 切换
  - 手机号、密码、验证码输入框
  - 获取验证码按钮（带倒计时功能）
  - 忘记密码链接（Toast 提示）
  - 保持与现有项目一致的科技感/工业风样式

### 2.3 状态管理
- 创建 `useAuth.ts` Hook 管理登录状态
- 在 App.tsx 中添加登录状态管理
- 替换现有用户信息显示为登录状态

### 2.4 集成到 App.tsx
- 添加登录按钮到侧边栏
- 通过状态控制登录/注册弹窗的显示与隐藏
- 实现登录成功后的状态切换

## 3. 技术细节

### 3.1 样式设计
- 主色调：#5B5FC7（紫色）
- 辅助色：#E0E7FF（浅紫色背景）
- 边框：1px solid #E5E9FF
- 圆角：8px-12px（与现有组件保持一致）
- 按钮样式：渐变背景或纯色背景，带 hover 效果

### 3.2 表单验证
- 手机号格式验证
- 密码长度验证（至少6位）
- 验证码格式验证（6位数字）
- 实时反馈验证结果

### 3.3 交互设计
- 登录/注册 Tab 切换动画
- 输入框聚焦效果
- 获取验证码倒计时（60秒）
- 登录/注册加载状态
- 错误提示 Toast

## 4. 实现顺序
1. 修改 `services/api.ts`，添加 Auth 相关接口
2. 创建 `hooks/useAuth.ts`，实现登录状态管理
3. 开发 `components/Auth/AuthModal.tsx` 组件
4. 修改 `App.tsx`，集成登录/注册功能
5. 测试所有功能，确保样式一致
